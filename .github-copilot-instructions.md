# Copilot Instructions for StixNVibes

## Project Overview

**StixNVibes** is a premium sticker store landing page built with React, Vite, and Tailwind CSS. The site features immersive interactive glass-morphism UI with mouse-tracking refraction effects, deep-space backgrounds with beams and starfields, and smooth animations via Framer Motion and React Three Fiber.

## Build, Test, and Lint Commands

### Development
```bash
npm run dev
```
Starts Vite dev server on http://localhost:3000 with hot module reloading.

### Production Build
```bash
npm run build
```
Creates optimized build in the `dist/` directory.

### Linting
```bash
npm lint
```
Runs ESLint across all `.js` and `.jsx` files using the flat config in `eslint.config.js`.

Note: There are no tests in this project currently. Linting is the only automated quality check.

## High-Level Architecture

### Directory Structure
```
src/
├── pages/              # Route-based pages (Landing, Contact, Polaroid, etc.)
├── components/         # Reusable UI components (Header, Footer, Effects, etc.)
├── data/              # Data layer with JSON files and utility functions
├── config/            # Configuration files
├── context/           # React Context for global state
├── lib/               # Helper functions and utilities
├── index.css          # Global styles
├── App.jsx            # Main router setup
└── main.jsx           # React entry point
```

### Technology Stack
- **React 18** with React Router v6 for routing
- **Vite** for fast development and optimized builds
- **Tailwind CSS** with custom theme (Plus Jakarta Sans font, custom colors)
- **Framer Motion** for smooth animations and transitions
- **React Three Fiber** & **Three.js** for 3D graphics
- **@react-three/cannon** for physics simulations
- **@react-three/drei** for 3D utilities

### Data Architecture
Data is centralized in `src/data/` with utility functions in `dataUtils.js`:
- **products.json** - Individual sticker products with categories, tags, popularity
- **collections.json** - Pre-curated product bundles
- **categories.json** - Category definitions for filtering
- **Data utilities** - Functions for fetching, filtering, and searching (getAllProducts, getProductsInCollection, getTrendingProducts, etc.)

Products support multiple collections and have properties: id, title, image, price, type (Vinyl/Normal/Waterproof/Polaroid), tags, category, popularity, description.

### Routing
App-level routes are defined in `src/App.jsx` using React Router:
- `/` → LandingPage
- `/polaroid` → PolaroidPage
- `/poster-series` → PosterSeriesPage
- `/single-posters` → SinglePostersPage
- `/submit-art` → SubmitArtPage
- `/contact` → ContactPage
- `*` → NotFound

Pages typically render collections of products or interactive displays. See MIGRATION_GUIDE.md for details on the enhanced data structure.

## Key Conventions

### Component Structure
- **Functional components only** - All components use React hooks (useState, useEffect, useContext, etc.)
- **Props over global state** - Data flows top-down via props; use React Context for truly global state
- **Export as default** - All components export as `export default ComponentName` (see Header.jsx, Footer.jsx)
- **Naming** - PascalCase for components, camelCase for functions/utilities

### Styling
- **Tailwind-first approach** - Style directly in JSX with Tailwind classes
- **Custom Tailwind tokens** - Use extended theme colors (primary: #e92932, dark, light, muted)
- **Glass morphism** - Use `bg-white/[0.03]`, `backdrop-blur-xl`, and `border-white/10` for premium glass effect
- **Responsive patterns** - Use Tailwind breakpoints (sm, md, lg); see Header.jsx for mobile-first patterns
- **Animation classes** - Custom keyframes are predefined in tailwind.config.js (marquee, shimmer, gradient-x, etc.)

### React Patterns
- **Mobile menu state** - Use `useState` to manage mobile menu open/close; see Header.jsx example
- **useLocation hook** - Use `useLocation()` from react-router-dom to determine active route for styling
- **AnimatePresence** - Wrap conditional renders with Framer Motion's `AnimatePresence` for exit animations
- **Motion components** - Use `motion.div`, `motion.section` for animated elements
- **Lazy state management** - If you need more complex state, consider creating a context file in `src/context/`

### Data Fetching & Utilities
- **Use dataUtils.js** - Import functions like `getAllProducts()`, `getProductsInCollection()`, `getTrendingProducts()` instead of directly importing JSON
- **Products have multiple collections** - Use the `collections` array on a product (previously was a single `collection` field)
- **Popularity sorting** - Products have a `popularity` score (0-100) used for trending; sort by this for "best sellers"
- **Material multipliers** - Price multipliers exist for material types; implement in cart/checkout logic if needed

### 3D Components
- React Three Fiber components use the `<Canvas>` element; check BackgroundEffects.jsx and similar for patterns
- Use `@react-three/drei` utilities (Effects, Stars, etc.) for reusable 3D primitives
- Avoid heavy 3D on mobile; use MobileOptimized.jsx pattern for device detection

### Layout & Responsive Design
- **Container** - Tailwind container with center: true and 1rem padding is configured
- **Grid helpers** - Custom grid-template-rows and grid-template-columns utilities (grid-rows-2, grid-cols-3, etc.) are available
- **Mobile detection** - Use breakpoint classes (hidden lg:flex) rather than JavaScript for most layouts
- **Marquee animations** - Custom marquee animation is defined in tailwind.config.js; see MarqueeComponent.jsx

### ESLint Rules
- **Unused variables** - Exceptions for capitalized or underscore-prefixed names (varsIgnorePattern: '^[A-Z_]')
- **React Hooks** - ESLint plugin enforces hooks rules of hooks
- **React Refresh** - Only export components from files, not constant exports (unless explicitly allowed)

## Important Codebase Rules & Patterns

1. **Data-first design** - Always use dataUtils.js for data fetching; do not import JSON directly in most components
2. **No test files currently** - Focus on linting with `npm run lint` if making significant changes
3. **Logo configuration** - Site-wide logo lives in `src/components/Logo.jsx`; update there once to reflect everywhere
4. **Deployment targets** - Both Netlify (netlify.toml) and Vercel (vercel.json) configs exist; ensure builds work with both
5. **Vite port** - Dev server defaults to port 3000 with auto-open; defined in vite.config.js

## When Adding New Features

- **New page?** Create a new file in `src/pages/` and add the route in `src/App.jsx`
- **New product/collection?** Add to the respective JSON in `src/data/` and use dataUtils to fetch
- **New reusable component?** Create in `src/components/` and export as default
- **New utility function?** Add to `src/lib/` or extend `src/data/dataUtils.js`
- **New animation?** Define keyframes in `tailwind.config.js` if reusable, or use Framer Motion inline
- **New color/font/spacing?** Extend the theme in `tailwind.config.js`
